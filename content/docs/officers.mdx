# Officers

This section provides information about the endpoints for managing officers.

## Create a new officer

**Endpoint:** `POST /v1/officers`

Creates a new officer.
**Permission Required:** `officers:write`

To make this request, you need to be authenticated and have the `officers:write` permission. Include your authentication token in the `Authorization` header.

**Request Body:**

```json
{
    "regulation_number": "12345",
    "first_name": "John",
    "last_name": "Doe",
    "sex": "Male",
    "rank_id": 1,
    "formation_id": 1,
    "posting_id": 1,
    "is_active": true
}
```

**Validation Rules:**

*   `regulation_number`: Must not be empty and must be less than or equal to 50 characters.
*   `first_name`: Must not be empty and must be less than or equal to 100 characters.
*   `last_name`: Must not be empty and must be less than or equal to 100 characters.
*   `sex`: Must be either `Male` or `Female`.

**Example Request:**

```bash
curl -X POST \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <TOKEN>" \
  -d '{
    "regulation_number": "12345",
    "first_name": "John",
    "last_name": "Doe",
    "sex": "Male",
    "rank_id": 1,
    "formation_id": 1,
    "posting_id": 1,
    "is_active": true
  }' "http://localhost:4000/v1/officers"
```

**Example Success Response (201 Created):**

```json
{
    "officer": {
        "id": 1,
        "regulation_number": "12345",
        "first_name": "John",
        "last_name": "Doe",
        "sex": "Male",
        "rank_id": 1,
        "formation_id": 1,
        "posting_id": 1,
        "is_active": true
    }
}
```

**Error Handling:**

*   If the request body is invalid or validation fails, you will receive a `422 Unprocessable Entity` response.

## Retrieve an officer

**Endpoint:** `GET /v1/officers/:id`

Retrieves a specific officer by their ID.
**Permission Required:** `officers:read`

To make this request, you need to be authenticated and have the `officers:read` permission. Include your authentication token in the `Authorization` header.

**Example Request:**

```bash
curl -H "Authorization: Bearer <TOKEN>" "http://localhost:4000/v1/officers/1"
```

**Example Success Response (200 OK):**

```json
{
    "officer": {
        "id": 1,
        "regulation_number": "12345",
        "first_name": "John",
        "last_name": "Doe",
        "sex": "Male",
        "rank_id": 1,
        "formation_id": 1,
        "posting_id": 1,
        "is_active": true
    }
}
```

**Error Handling:**

*   If the officer with the specified ID does not exist, you will receive a `404 Not Found` response.

## Update an officer

**Endpoint:** `PATCH /v1/officers/:id`

Updates a specific officer's details.
**Permission Required:** `officers:write`

To make this request, you need to be authenticated and have the `officers:write` permission. Include your authentication token in the `Authorization` header.

**Request Body (Partial):**

```json
{
    "first_name": "Jane"
}
```

**Example Request:**

```bash
curl -X PATCH \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <TOKEN>" \
  -d '{
    "first_name": "Jane"
  }' "http://localhost:4000/v1/officers/1"
```

**Example Success Response (200 OK):**

```json
{
    "officer": {
        "id": 1,
        "regulation_number": "12345",
        "first_name": "Jane",
        "last_name": "Doe",
        "sex": "Male",
        "rank_id": 1,
        "formation_id": 1,
        "posting_id": 1,
        "is_active": true
    }
}
```

**Error Handling:**

*   If the officer does not exist, you will receive a `404 Not Found` response.
*   If validation fails, you will receive a `422 Unprocessable Entity` response.
*   If there is an edit conflict, you will receive a `409 Conflict` response.

## Delete an officer

**Endpoint:** `DELETE /v1/officers/:id`

Deletes a specific officer.

To make this request, you need to be authenticated and have the `officers:write` permission. Include your authentication token in the `Authorization` header.

## List all officers

**Endpoint:** `GET /v1/officers`

Retrieves a paginated list of all officers.
**Permission Required:** `officers:read`

To make this request, you need to be authenticated and have the `officers:read` permission. Include your authentication token in the `Authorization` header.

**Query Parameters:**

*   `page` (integer): The page number to retrieve. Defaults to `1`.
*   `page_size` (integer): The number of records to retrieve per page. Defaults to `20`, with a maximum of `100`.
*   `sort` (string, optional): The field to sort by (e.g., `first_name`, `-first_name`).

**Example Request:**

```bash
curl -H "Authorization: Bearer <TOKEN>" "http://localhost:4000/v1/officers?page=1&page_size=5"
```

**Example Success Response (200 OK):**

```json
{
    "officers": [
        {
            "id": 1,
            "regulation_number": "12345",
            "first_name": "John",
            "last_name": "Doe",
            "sex": "Male",
            "rank_id": 1,
            "formation_id": 1,
            "posting_id": 1,
            "is_active": true
        }
    ],
    "metadata": {
        "current_page": 1,
        "page_size": 5,
        "first_page": 1,
        "last_page": 1,
        "total_records": 1
    }
}
```

**Error Handling:**

*   If you provide invalid query parameters (e.g., `page=-1`), you will receive a `422 Unprocessable Entity` response with validation errors.
